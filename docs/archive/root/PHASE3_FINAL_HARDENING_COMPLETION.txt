ENDPOINT SPECIFICATION
================================================================================

Route: DELETE /session/{session_id}

Authentication: ✅ JWT Bearer token required
Authorization: ✅ User ownership validated (user_id check)

Request:
  - session_id: Integer (path parameter)

Response (200 OK):
  {
    "message": "Session deleted successfully",
    "session_id": 123,
    "deleted_records": {
      "answers": 5,
      "behavior_metrics": 3,
      "behavior_issues": 2
    },
    "timestamp": "2026-04-14T15:30:45.123456+00:00"
  }

HTTP Status Codes:
  - 200 OK: Deletion successful
  - 400 Bad Request: Session is active (must end or abandon first)
  - 404 Not Found: Session not found or user doesn't own session
  - 409 Conflict: Answer submission in progress (concurrent). Wait and retry.
  - 500 Internal Server Error: Database error (automatic rollback)

Cascade Behavior:
  - InterviewAnswer: Deleted ✅
  - BehaviorIssue: Deleted ✅
  - InterviewBehaviorMetric: Deleted ✅

Transaction Safety:
  - Atomic: All-or-nothing semantics ✅
  - Rollback on exception: Automatic ✅
  - Race condition safe: Yes ✅
